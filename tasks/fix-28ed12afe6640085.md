# fix-28ed12afe6640085 — E1T1: /search flow is a dead-end — stored place is never consumed

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 SATB

The `/search` command sets `session.step = "awaiting_place"`, prompts for a place name, stores it in `session.place`, replies `"Place saved: ..."`, and clears the step. No handler ever reads `session.place`. While E1T1 technically fulfills its literal spec ("store the input in session state"), the guided flow ends with dead data — typing `/search` → `London` produces `"Place saved: London"` and nothing else. The E1 epic's intended search→geocode journey is broken: E1T2 and E1T3 each define their own separate commands (`/whereis`, `/geocode`) with independent step flows that never read E1T1's stored `session.place`.
