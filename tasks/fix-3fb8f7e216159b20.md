# fix-3fb8f7e216159b20 — E1T2/E1T3: inline geocoding result buttons have no callback_query handler

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 SATB

Both `src/handlers/E1T2.ts:59-62` and `src/handlers/E1T3.ts:93-96` create inline keyboard buttons with `callback_data` of the form `geocode:<lat>:<lon>`. However, neither handler (nor any handler in `src/handlers/`) registers a grammY `callback_query` listener for the `geocode:` prefix. When a user taps a result button, the callback query is sent to the bot but never matched — leaving the user with no response. The fallback `bot.on("message", ...)` at `src/bot.ts:48` only handles messages, not callback queries, so the press is silently dropped.
