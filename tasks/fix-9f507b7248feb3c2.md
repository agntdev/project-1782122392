# fix-9f507b7248feb3c2 — Admin error notification misses errors from 18 of 23 handlers due to alphabetical load order

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 SATB

`buildBot()` loads handlers in alphabetical filename order via `bot.use()`. E8T1 sorts 19th out of 23 handlers. The middleware wraps `next()` and only catches errors from handlers loaded **after** it in the middleware chain (E9T1, T01, T03, start, and the fallback handler — 5 of 23). Errors thrown by the 18 handlers that sort before E8T1 (E1T1 through E7T2) bypass the admin notification entirely and fall directly through to the global `bot.catch` handler, which only does `console.error` (no admin notification). The `createBot` function in `src/toolkit/bot.ts:47` supports an `onError` callback that would catch ALL errors if wired in, but the spec prohibits editing `src/bot.ts` to pass it. This means most handler errors will never be reported to the admin, defeating much of the feature's purpose.
