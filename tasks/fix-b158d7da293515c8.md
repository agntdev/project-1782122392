# fix-b158d7da293515c8 — Localization is never applied to any bot message — dead code

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 SATB

E9T1 defines `translate()`, `getLang()`/`setLang()`, and a translations dictionary mapping ~20 English strings to Russian, plus a `/lang` command that persists the user's language choice. However, **no other handler** (`start.ts`, `T01.ts`, `T03.ts`, `E1T1.ts`, `E1T2.ts`, `E6T1.ts`, `E6T2.ts`) imports or calls `translate()` or `getLang()`. The fallback in `bot.ts:48` also sends untranslated English. All bot messages are served in hardcoded English regardless of the `/lang` setting. The core spec requirement "Implement localization for all bot messages and UI elements in Russian" is not met — the localization system has zero effect on user-visible output. The `/lang` command itself translates its own confirmations inline (hardcoded), confirming the infrastructure works, but it is never wired into the bot's message pipeline. Fix: E9T1.ts needs a grammY middleware that intercepts all outgoing `ctx.reply()` text (and menu button labels) and runs them through `translate()`.
